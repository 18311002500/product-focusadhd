# FocusFlow - ADHD生产力工具产品需求文档 (PRD)

## 1. 产品概述

### 产品名称
**FocusFlow**（中文名：专注流）

### 核心价值主张
FocusFlow是一款专为ADHD（注意力缺陷多动障碍）患者设计的生产力工具。针对传统任务管理工具对ADHD大脑"无效"的痛点，FocusFlow通过游戏化机制、即时正向反馈和身体活动整合，帮助ADHD用户建立可持续的任务完成习惯。

**一句话定位**：让ADHD大脑也能享受完成任务的多巴胺奖励

### 产品愿景
让每一个注意力困难者都能找到属于自己的节奏，把"做不到"变成"已完成"。

---

## 2. 目标用户画像

### 主要用户群体
1. **确诊ADHD成年人**（25-40岁）
   - 已被确诊，正在寻找适合自己的生产力工具
   - 付费意愿高，对ADHD友好设计敏感
   - 尝试过传统Todo App但效果不佳

2. **疑似ADHD / 注意力困难者**（20-35岁）
   - 感觉自己"无法专注"、"总是拖延"
   - 在社交媒体上看到ADHD相关内容后自我怀疑
   - 希望通过工具改善注意力问题

3. **ADHD青少年家长**（为孩子寻找工具）
   - 希望帮助孩子建立任务完成习惯
   - 寻求游戏化、有趣的解决方案

### 用户核心痛点
- ❌ 传统Todo List带来挫败感而非成就感
- ❌ 长时间专注（25分钟番茄钟）对ADHD太难
- ❌ 缺乏即时反馈，动力难以维持
- ❌ 久坐不动加剧注意力涣散
- ❌ "要么全做要么不做"的极端思维模式

## 4. 游戏化机制设计

### 4.1 能量点系统
- **简单任务**：+10 能量点
- **中等任务**：+20 能量点
- **困难任务**：+50 能量点
- **连续完成加成**：连续7天+10%加成

### 4.2 徽章体系
| 徽章名称 | 获得条件 | 图标 |
|---------|---------|------|
| 初学者 | 完成第一个任务 | 🌱 |
| 连续战士 | 连续3天完成任务 | 🔥 |
| 专注大师 | 累计专注10小时 | ⏰ |
| 任务终结者 | 完成50个任务 | ⚔️ |
| 早起鸟 | 上午9点前完成任务 | 🐦 |
| 夜晚战士 | 晚上完成任务 | 🌙 |
| 身体活动家 | 结合运动完成任务 | 🏃 |
| 完美一周 | 一周每天完成任务 | 🏆 |

### 4.3 成长系统
- 用户拥有一株虚拟植物（或宠物）
- 完成任务为植物提供"能量"
- 植物随积分增长而成长、开花、结果
- 多日不完成任务植物会"枯萎"（温和提醒）

## 5. 用户流程图

### 5.1 新用户引导流程
```
启动App → 欢迎页 → 选择用户类型（ADHD/疑似/家长）
                                    ↓
设置每日目标 → 创建第一个任务 → 开始首次专注 → 完成引导徽章
```

### 5.2 日常任务流程
```
打开App → 查看今日任务池 → 选择任务 → 开始专注计时
                                    ↓
              完成任务 ← 获得积分/徽章 ← 专注完成
```

### 5.3 核心交互流程
```
主页
├── 今日任务区（大按钮卡片）
│   ├── 点击开始 → 专注计时页
│   └── 长按编辑 → 编辑/删除
├── 进度概览（本周完成率）
└── 我的植物（成长状态）

专注计时页
├── 倒计时显示（大数字）
├── 进度环（彩色）
├── 暂停/放弃/完成按钮
└── 背景音开关

完成页
├── 获得积分动画
├── 徽章解锁提示
├── 植物成长动画
└── 分享按钮（可选）
```

## 6. 技术架构

### 6.1 技术栈
- **开发语言**：Swift 5.9+
- **UI框架**：SwiftUI
- **最低版本**：iOS 16.0
- **数据持久化**：Core Data + UserDefaults
- **健康数据**：HealthKit
- **通知**：UserNotifications

### 6.2 架构模式：MVVM
```
┌─────────────────────────────────────────────┐
│                    VIEW                     │
│  (SwiftUI Views - 声明式UI，响应状态变化)       │
└──────────────────┬──────────────────────────┘
                   │ @Published / @StateObject
                   ▼
┌─────────────────────────────────────────────┐
│                 VIEWMODEL                   │
│  (ObservableObject - 业务逻辑，状态管理)        │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│                  MODEL                      │
│  (Core Data + 领域模型 - 数据结构定义)          │
└─────────────────────────────────────────────┘
```

### 6.3 数据模型

#### Core Data 实体
```swift
// Task 任务实体
Task
├── id: UUID
├── title: String
├── notes: String
├── difficulty: Int16 (0=简单, 1=中等, 2=困难)
├── isCompleted: Bool
├── createdAt: Date
├── completedAt: Date?
├── dueDate: Date?
├── subtasks: [Subtask] (关系)
└── tags: [String] (Transformable)

// FocusSession 专注记录
FocusSession
├── id: UUID
├── startTime: Date
├── endTime: Date?
├── duration: Int16 (计划时长)
├── actualDuration: Int16 (实际时长)
├── taskId: UUID?
└── wasCompleted: Bool

// Badge 徽章
Badge
├── id: String
├── name: String
├── description: String
├── icon: String
├── isUnlocked: Bool
├── unlockedAt: Date?
└── progress: Double

// UserStats 用户统计
UserStats
├── totalPoints: Int32
├── currentStreak: Int16
├── longestStreak: Int16
├── totalTasksCompleted: Int32
├── totalFocusMinutes: Int32
└── plantGrowthLevel: Int16
```

### 6.4 本地存储策略
- **Core Data**：任务、专注记录、徽章、用户统计
- **UserDefaults**：用户偏好设置、首次启动标记
- **App Groups**（未来扩展）：iCloud同步准备

### 6.5 零服务器成本设计
| 功能 | 实现方式 |
|------|----------|
| 数据存储 | Core Data (本地SQLite) |
| 用户账户 | 无需登录，设备本地使用 |
| 数据备份 | iCloud自动备份（用户自行开启） |
| 通知 | 本地通知，无需APNs服务器 |
| 健康数据 | HealthKit本地读取 |
| 更新 | App Store更新 |

## 7. UI设计规范

### 7.1 设计理念
- **减少认知负荷**：大按钮、简洁文字、清晰层级
- **正向情绪**：明亮色彩、圆润形状、庆祝动画
- **即时反馈**：点击有响应、进度可视化

### 7.2 色彩方案
| 用途 | 颜色 | Hex |
|------|------|-----|
| 主色 | 活力橙 | #FF6B35 |
| 次色 | 清新绿 | #4ECDC4 |
| 强调色 | 阳光黄 | #FFE66D |
| 背景色 | 浅灰白 | #F7F7F7 |
| 文字主色 | 深灰 | #2C3E50 |
| 成功色 | 翠绿 | #27AE60 |
| 困难任务 | 粉红 | #E91E63 |

### 7.3 字体规范
- **标题**：SF Pro Display Bold / 28-34pt
- **正文**：SF Pro Text Regular / 16-18pt
- **按钮文字**：SF Pro Text Semibold / 18-20pt
- **大数字（计时器）**：SF Pro Display Bold / 72pt

### 7.4 组件规范
- **任务卡片**：圆角20pt，阴影，高度100pt
- **主按钮**：圆角16pt，填充色，高度60pt
- **计时器**：圆形进度环，线宽12pt
- **徽章图标**：60x60pt，圆角背景

## 8. 定价策略

### 8.1 付费模式
采用**双重选择**模式，满足不同用户需求：

| 方案 | 价格 | 功能 |
|------|------|------|
| **终身版** | $9.99 | 解锁所有功能，永久使用 |
| **订阅版** | $2.99/月 | 全部功能，按月付费 |

### 8.2 免费版限制
- 最多创建10个任务
- 基础专注计时器
- 3个基础徽章
- 基础进度统计

### 8.3 付费解锁功能
- 无限任务数量
- 高级专注统计与热力图
- 全部20+徽章
- 虚拟植物成长系统
- 自定义主题色彩
- 数据导出功能
- iCloud同步（未来版本）

### 8.4 定价依据
- 竞品Forest售价$3.99（一次性）
- Todoist订阅$4/月
- ADHD专用工具较少，差异化定价
- $9.99终身版降低用户决策门槛
